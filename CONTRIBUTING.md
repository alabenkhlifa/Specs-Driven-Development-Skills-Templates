# Contributing

Contributions should keep the starter kit small, tool-neutral at its core, and strict about workflow boundaries.

Before opening a pull request:

1. Update the canonical workflow under `skills/` or the canonical resource under `templates/`.
2. Update its command mirror, adapter, link, or bundled copy when applicable.
3. Keep examples free of private project information.
4. Run `python3 scripts/test_validate_spec.py` and `python3 scripts/validate_repo.py`.
5. Explain the workflow problem the change solves.

Avoid adding a new template section unless it preserves a decision, constrains implementation, or defines proof.
